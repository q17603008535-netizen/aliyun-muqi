## 项目背景

### 现状分析

- **项目名称**: 洛天翡翠网站 V3 (jade-website-v3)
- **当前状态**: 已上线运营，正在投放 Google Ads
- **技术栈**: Astro + Tailwind CSS + TypeScript
- **部署平台**: Vercel（免费额度）
- **域名**: https://ltmdfc.top

### 存在问题

根据 Gemini 分析报告和实际运营反馈：

1. **SEO 内容不足**
   - 网站文字太少，缺乏深度内容
   - 关键词（如"缅甸翡翠原石"）在 H1-H3 和 meta 描述中严重不足
   - Google Ads 质量得分低，导致点击成本高

2. **转化追踪不精准**
   - 目前 GTM 只能追踪页面浏览
   - 无法追踪"一键复制微信"等关键转化行为
   - Google Ads 无法优化转化，投放效果受限

3. **单页架构限制**
   - 所有产品都在首页，广告落地页相关性低
   - 用户搜索"莫西沙原石"却 landing 到首页，需要自己寻找内容
   - Google 判定页面相关性低，扣除质量得分

### 迭代目标

**核心目标**: 通过增加科普知识库，提升 SEO 和广告效果

**具体指标**:
- ✅ 增加科普内容，每个场口专题 800-1500 字
- ✅ 优化关键词布局（H1-H3 + meta）
- ✅ 实现精准转化追踪（复制微信行为）
- ✅ 保持 Vercel 免费额度内
- ✅ **保证现有功能不崩溃**（最高优先级）

## 需求分析

### 功能需求

#### 1. 首页科普窗口（双入口之一）

**需求描述**:
- 在首页展示科普窗口（卡片式）
- 展示 3 篇精选科普文章（带封面图、标题、描述）
- 每篇文章卡片可点击 → 跳转到单篇详情页
- 板块旁边有"浏览全部科普"按钮 → 跳转到总科普页

**优先级**: P0（核心功能）

#### 2. 总科普页（文章列表页）

**需求描述**:
- 路径：`/knowledge/`
- 顶部导航栏入口（全站可访问）
- 展示所有科普文章列表
- 支持按分类筛选（场口解析、选购技巧、避坑指南）
- 点击任意文章 → 跳转到单篇详情页

**优先级**: P0（核心功能）

#### 3. 单篇科普文详情页

**需求描述**:
- 路径：`/knowledge/[slug]/`
- 顶部导航栏（可返回首页、总科普页）
- 完整的科普文章内容（Markdown 渲染）
- 支持图片展示
- 返回按钮 → 返回总科普页
- 上一篇/下一篇导航
- 文章末尾添加"复制微信"按钮（带转化追踪）

**优先级**: P0（核心功能）

#### 4. 后台内容管理

**需求描述**:
- 通过编辑 Markdown 文件管理内容
- 支持增删改查科普文章
- 支持上传图片
- 支持设置"精选文章"（在首页展示）
- 支持设置分类、标签

**优先级**: P0（核心功能）

## 架构设计

### 整体架构

```
┌─────────────────────────────────────────────┐
│              用户访问层                      │
├─────────────────────────────────────────────┤
│  首页  │  总科普页  │  单篇详情页  │  其他页面  │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│              组件层                          │
├─────────────────────────────────────────────┤
│  KnowledgeWindow  │  KnowledgeCard  │  Nav  │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│          Content Collections（数据层）        │
├─────────────────────────────────────────────┤
│  src/content/knowledge/*.md（Markdown 文件）  │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│           静态站点生成（Astro）               │
├─────────────────────────────────────────────┤
│  构建时渲染 → 纯 HTML 文件 → Vercel 托管      │
└─────────────────────────────────────────────┘
```

### 技术架构决策

| 决策点 | 选项 | 选择 | 理由 |
|--------|------|------|------|
| **内容管理** | CMS / Markdown / Headless CMS | **Markdown** | 零成本、零维护、Vercel 原生支持 |
| **页面渲染** | SSR / SSG / ISR | **SSG** | 静态预渲染，性能最优 |
| **图片管理** | 第三方 CDN / 本地托管 | **本地托管** | 免费、可控、简单 |
| **导航组件** | 独立组件 / 复用现有 | **复用现有** | 保持一致性，减少开发量 |

## 数据流设计

### Content Collection Schema

```typescript
// src/content/config.ts
import { defineCollection, z } from 'astro:content';

const knowledgeCollection = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),                    // 文章标题
    description: z.string(),              // 描述（SEO）
    publishDate: z.date(),                // 发布日期
    category: z.enum([                   // 分类
      '场口解析',
      '选购技巧',
      '避坑指南',
      '玉石文化'
    ]),
    tags: z.array(z.string()),           // 标签
    image: z.string().optional(),        // 封面图
    featured: z.boolean().default(false), // 是否精选（首页展示）
    order: z.number().default(0),        // 排序权重
  }),
});

export const collections = {
  'knowledge': knowledgeCollection,
};
```


## 转化追踪

### 追踪逻辑

#### 现状

- ✅ GTM 已配置
- ✅ 可追踪页面浏览
- ❌ 无法追踪"复制微信"行为

#### 升级方案

**采用 Gemini 推荐的混合方案**:

```javascript
// 在复制按钮的点击事件中
async function handleCopy() {
  await navigator.clipboard.writeText(wechatId);
  
  // 方案 A：直接 gtag 调用（保证追踪）
  try {
    if (typeof gtag === 'function') {
      gtag('event', 'conversion', {
        'send_to': 'AW-CONVERSION_ID/CONVERSION_LABEL'
      });
    }
  } catch (e) {
    console.warn('gtag conversion failed:', e);
  }
  
  // 方案 C：dataLayer 推送（扩展性）
  window.dataLayer = window.dataLayer || [];
  dataLayer.push({
    event: 'copy_wechat',
    timestamp: new Date().toISOString(),
    source: window.location.pathname,
    article: '莫西沙场口全解析'  // 可追踪具体文章
  });
}
```

### 实施步骤

1. **获取 Google Ads 转化 ID**
   - 路径：Google Ads 后台 > 目标 > 转化 > 网页设置
   - 格式：`AW-XXXXXXXXX/YYYYYYY`

2. **修改复制按钮组件**
   - 文件：`src/components/FloatingContact.astro`
   - 添加 gtag 调用逻辑

3. **测试验证**
   - 使用 Google Tag Assistant
   - 在 Google Ads 后台查看转化数据

## 风险评估与应对

### 高风险项

#### 1. 现有功能崩溃（最高优先级）

**风险**: 修改首页和导航栏可能影响现有功能

**应对措施**:
- ✅ 渐进式修改，不影响其他组件
- ✅ 本地充分测试后再部署
- ✅ 准备回滚方案
- ✅ 部署后立即验证所有功能

#### 2. 性能下降

**风险**: 增加页面可能导致加载变慢

**应对措施**:
- ✅ 严格遵循性能优化策略
- ✅ Lighthouse 测试达标后再部署
- ✅ 图片懒加载 + 优化

#### 3. SEO 负面影响

**风险**: 新页面可能稀释现有 SEO 权重

**应对措施**:
- ✅ 合理的内部链接结构

### 中风险项

#### 1. 内容管理复杂度

**风险**: Markdown 文件管理可能不如 CMS 直观
