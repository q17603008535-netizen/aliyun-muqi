## 🚨 错误诊断

### Vercel 部署错误信息
```
[vite:build-import-analysis] [plugin vite:build-import-analysis] 
astro:scripts/page-ssr.js (1:1852): Failed to parse source for import analysis 
because the content contains invalid JS syntax.
```

### 根本原因

**问题包**：`astro-i18next` (v1.0.0-beta.21)

**技术原因**：
1. `astro-i18next` 包与 Astro v5 不兼容
2. 该包尝试将翻译配置（包含中文的 JSON 对象）作为 JavaScript 代码解析
3. 翻译文本中的换行符和特殊字符导致 Rollup 解析失败

**错误堆栈**：
```
at getRollupError (rollup/dist/es/shared/parseAst.js:402:41)
at Object.error (rollup/dist/es/shared/node-entry.js:22249:20)
at Object.handler (vite/dist/node/chunks/dep-D4NMHUTW.js:45418:16)
```

## ✅ 解决方案

### 方案选择

**选择**：移除 `astro-i18next`，使用纯前端原生实现

**理由**：
- ✅ 完全兼容 Astro v5
- ✅ 无额外依赖，包体积更小
- ✅ 更灵活的控制
- ✅ 性能更好（客户端直接切换，无需服务端渲染）

### 实施步骤

#### 1. 移除不兼容包
```bash
npm uninstall astro-i18next
rm astro-i18next.config.ts
```

#### 2. 更新 Astro 配置
修改 `astro.config.mjs`：
- 移除 `astro-i18next` 集成
- 保留 `@astrojs/sitemap`

#### 3. 创建原生 i18n 模块

**新增文件**：
- `src/i18n/translations.ts` - 翻译配置
- `src/i18n/index.ts` - 客户端工具函数

**功能实现**：
```typescript
// 翻译配置（3 种语言）
export const translations: Translations = {
  "zh-TW": { ... },  // 繁体中文
  "zh-CN": { ... },  // 简体中文
  "en": { ... }      // 英文
};

// 国家到语言映射
export const countryToLocale: Record<string, string> = {
  CN: 'zh-CN', TW: 'zh-TW', HK: 'zh-TW',
  US: 'en', GB: 'en', CA: 'en', AU: 'en', NZ: 'en'
};
```

#### 4. 更新组件

**LanguageSwitcher.astro**：
- 移除对 `astro-i18next` 的依赖
- 使用原生语言切换逻辑
- 通过 URL 路由切换语言

**src/scripts/init.ts**：
- 更新导入路径
- 使用新的 `initAutoLocaleSwitch`
